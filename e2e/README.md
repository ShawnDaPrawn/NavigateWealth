# Playwright E2E (P8.9) — opt-in

This folder contains end-to-end happy-path tests for the e-sign module.
They are **deliberately excluded** from `npm test` (Vitest) because they
require:

1. A live dev server. Default base URL is `http://localhost:5173`.
   Override with `PLAYWRIGHT_BASE_URL=...`.
2. A reachable Supabase instance — the specs hit the real `/esign`
   endpoints and so cannot be replayed against fixtures alone.
3. Browser binaries installed via `npx playwright install`.

## Running locally

```bash
# 1. start the app
npm run dev

# 2. install browsers (first time only)
npx playwright install --with-deps chromium

# 3. set up env so tests can sign in / fetch a signing token
cp e2e/.env.example .env.local   # then edit credentials
export $(cat .env.local | xargs) # or use direnv

# 4. run all e2e specs (desktop + mobile viewports)
npm run test:e2e

# helpful flags
npm run test:e2e -- --headed                  # see the browser
npm run test:e2e -- --project=desktop-chromium
npm run test:e2e -- e2e/signer-decline.spec.ts
```

## Required env

| Variable | Purpose |
| --- | --- |
| `PLAYWRIGHT_BASE_URL` | Base URL for the running app (default `http://localhost:5173`) |
| `E2E_ADMIN_EMAIL`     | Admin user used by the *sender* spec to log in |
| `E2E_ADMIN_PASSWORD`  | Admin password |
| `E2E_SIGNER_TOKEN`    | A pre-generated, single-use signer token used by the *signer* and *decline* specs. Generated by sending a draft envelope from a seed account; copy the token from the magic-link in the activity log |
| `E2E_SIGNER_DECLINE_TOKEN` | Same as above but a separate envelope so the decline test can run independently |

If any required env var is missing the affected spec will be **skipped**
(not failed). This keeps the suite resilient to partial setups.

## Why opt-in?

Phase 8 of the production-grade roadmap explicitly defers CI wiring
until a stable ephemeral preview environment exists. Until then the
specs serve as a runnable QA checklist that engineers can execute
locally before each release.
